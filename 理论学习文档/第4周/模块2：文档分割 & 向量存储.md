## 🧩 第一步：文档分割 – `RecursiveCharacterTextSplitter`

在 RAG 中，不按页面拆分，而是按语义块来组织文档，能显著提升检索的精准度。

### 核心概念：chunk_size 与 chunk_overlap

*   `chunk_size`：定义每个文本块的最大长度（通常是字符数或Token数）。
*   `chunk_overlap`：定义相邻两个文本块之间共享的长度，用于保留上下文信息，避免关键信息被分割在块边界上导致丢失。

### 为什么选择 `RecursiveCharacterTextSplitter`？

它会按照一个预定义的字符列表（如 `["\n\n", "\n", " ", ""]`）进行递归分割：优先尝试用段落分隔符 `\n\n` 进行分割；如果切出来的块仍超过 `chunk_size`，会降级到下一级分隔符（如 `\n`），依此类推。这种方式能在保持文本自然语义连贯性的同时，灵活地控制块的大小。

### 参数调优参考

*   **起步建议**：可以从 `chunk_size = 500`、`chunk_overlap = 50` 开始，这对于大多数中文RAG场景是一个不错的基线。
*   **依据模型调整**：根据你的 Embedding 模型（阿里 `text-embedding-v4`）和最终使用的 LLM（DeepSeek）的上下文窗口大小进行调整。`text-embedding-v4` 单次调用文本长度上限是 8192 tokens，但为保障检索质量，建议单个块控制在 512~1024 tokens 之间。
*   **重叠量的平衡**：`chunk_overlap` 过小会丢失边界上下文，过大则会引入大量重复内容。通常设置为 `chunk_size` 的 10%-20% 是一个比较稳妥的选择。

## 🔗 第二步：阿里云 `text-embedding-v4` 接入

在 LangChain 中集成阿里云的 Embedding 模型主要有两种方式。推荐你使用**第二种（DashScopeEmbeddings）**，它更简洁且是官方维护的。

### 方式一：使用 `DashScopeEmbeddings`（推荐）

这是 LangChain 社区库中官方支持阿里云 DashScope 平台的 Embeddings 类，无需手动实现接口。

1.  **安装依赖**：
    ```bash
    pip install langchain-community dashscope
    ```

2.  **配置 API Key**：
    为安全起见，请将你的 DashScope API Key 配置为环境变量。代码会自动从该变量读取。

    ```python
    import os
    os.environ["DASHSCOPE_API_KEY"] = "你的-阿里云-dashscope-api-key"
    ```

3.  **代码实现**：
    通过 `DashScopeEmbeddings` 初始化 `text-embedding-v4` 模型，关键参数是 `model` 和 `dimension`。

    ```python
    from langchain_community.embeddings import DashScopeEmbeddings
    
    # 初始化 text-embedding-v4 模型，指定维度为 1024
    embeddings = DashScopeEmbeddings(
        model="text-embedding-v4",
        dimension=1024  # v4 支持 1024/1536/2048 等维度
    )
    
    # 示例：单条查询转换
    query_vector = embeddings.embed_query("这个文档主要讲了什么？")
    print(f"查询向量维度: {len(query_vector)}")
    ```

### 方式二：自定义封装（仅供了解）

你也可以通过 `OpenAI` 客户端的方式，手动继承 `Embeddings` 基类来实现封装。这种方式更灵活，但代码量较多。不过，官方推荐的前者 `DashScopeEmbeddings` 已经足够应对大部分场景，这里就不展开了。

## 💾 第三步：向量存储 – Chroma 持久化

Chroma 是一个轻量级、嵌入式的向量数据库，非常适合学习和开发。

### 安装依赖

```bash
pip install langchain-chroma chromadb
```

### 代码实现：从文档创建持久化的向量库

核心是使用 `Chroma.from_documents()` 方法，它会接收我们上一步生成的文档块列表 `chunks` 和 Embedding 模型 `embeddings`，自动完成向量化并存入数据库。

```python
from langchain_chroma import Chroma

# persist_directory 指定持久化存储的本地路径
persist_directory = "./chroma_db"

# 创建并持久化向量库
vectordb = Chroma.from_documents(
    documents=chunks,          # 上一步分割好的文档块列表
    embedding=embeddings,      # 上一步初始化的 Embeddings 模型
    persist_directory=persist_directory  # 关键参数：持久化目录
)

print(f"向量库创建完成，共存储 {vectordb._collection.count()} 个文档块")
```

### 关键操作

*   **后续加载**：当程序再次启动时，可以直接从本地目录加载已有的向量库，无需重复计算 embedding，可以大大节省时间和 API 调用开销。
    ```python
    vectordb = Chroma(
        persist_directory=persist_directory,
        embedding_function=embeddings
    )
    ```
*   **转换为检索器**：在构建 LCEL 链之前，通常需要将向量库转换为检索器 `retriever`，以便后续使用。
    ```python
    retriever = vectordb.as_retriever(search_kwargs={"k": 3})
    ```

## 🧪 第四步：完整代码演示

下面是一个整合了从 PDF 加载、分割、到创建向量库的完整脚本，你可以直接运行测试。

```python
# module2_load_split_store.py
import os
from langchain_community.document_loaders import PyPDFLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_community.embeddings import DashScopeEmbeddings
from langchain_chroma import Chroma

# ----------------------------
# 1. 配置 API Key
# ----------------------------
# 请将你的 DashScope API Key 配置在环境变量中
os.environ["DASHSCOPE_API_KEY"] = "你的-阿里云-dashscope-api-key"

# ----------------------------
# 2. 加载 PDF 文档
# ----------------------------
pdf_path = "./data/sample.pdf"  # 替换为你的 PDF 路径
loader = PyPDFLoader(pdf_path)
pages = loader.load()
print(f"PDF 加载完成，共 {len(pages)} 页。")

# ----------------------------
# 3. 分割文档
# ----------------------------
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,      # 每块最大 500 字符
    chunk_overlap=50,    # 块间重叠 50 字符
    separators=["\n\n", "\n", "。", " ", ""]  # 可自定义分隔符优先级
)
chunks = text_splitter.split_documents(pages)
print(f"文档已分割为 {len(chunks)} 个文本块。")

# ----------------------------
# 4. 初始化 Embedding 模型 (阿里云 text-embedding-v4)
# ----------------------------
embeddings = DashScopeEmbeddings(
    model="text-embedding-v4",
)

# ----------------------------
# 5. 创建并持久化 Chroma 向量库
# ----------------------------
persist_directory = "./chroma_db"
vectordb = Chroma.from_documents(
    documents=chunks,
    embedding=embeddings,
    persist_directory=persist_directory
)

print(f"向量库创建完成！共存储 {vectordb._collection.count()} 个向量。")
print(f"数据已持久化至目录: {persist_directory}")

# ----------------------------
# 6. 测试检索（可选）
# ----------------------------
query = "这个文档主要讲了什么内容？"
retrieved_docs = vectordb.similarity_search(query, k=2)
print(f"\n检索结果预览（共 {len(retrieved_docs)} 条）:")
for i, doc in enumerate(retrieved_docs):
    print(f"--- 结果 {i+1} ---")
    print(doc.page_content[:200] + "...")
```

## 💡 第五步：核心参数说明

*   **`chunk_size`**：决定了一个文本块的大小。较小的 `chunk_size` 能提高检索的颗粒度，但可能丢失上下文；较大的 `chunk_size` 则能保留更多信息，但可能引入噪声或超出模型限制。
*   **`chunk_overlap`**：让相邻块之间共享部分内容。例如，`chunk_size=500, chunk_overlap=50`，意味着第一个块包含字符 1-500，第二个块包含 451-950，依此类推。
*   **`dimension`（维度）**：`text-embedding-v4` 支持 1024、1536、2048 等维度，更高的维度通常能提供更强的语义表示能力，但也会增加存储和计算开销。
