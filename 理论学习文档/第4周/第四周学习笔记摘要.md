## 一、RAG 核心流程图

        │                                               │

## 二、关键代码资产（完整 RAG 问答脚本）

下面是一个**开箱即用**的脚本，整合了本周所有核心知识点。  
- 文档加载 → 分割 → 嵌入 → 存储 → 检索 → 生成  
- 使用阿里云 `text-embedding-v4` 作为嵌入模型  
- 使用 DeepSeek 作为对话模型  
- 使用 Chroma 本地持久化向量库  

```python
# rag_complete.py
import os
from dotenv import load_dotenv
from langchain_community.document_loaders import PyPDFLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_community.embeddings import DashScopeEmbeddings
from langchain_chroma import Chroma
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnablePassthrough, RunnableLambda

# ==================== 1. 配置 API Keys ====================
load_dotenv()

# ==================== 2. 加载 PDF 文档 ====================
pdf_path = "./data/sample.pdf"        # 请替换为你的 PDF 路径
loader = PyPDFLoader(pdf_path)
pages = loader.load()
print(f"✅ 加载完成，共 {len(pages)} 页")

# ==================== 3. 文档分割 ====================
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,        # 每块最大字符数
    chunk_overlap=50,      # 块间重叠字符数
    separators=["\n\n", "\n", "。", " ", ""]
)
chunks = text_splitter.split_documents(pages)
print(f"✅ 分割完成，共 {len(chunks)} 个文本块")

# ==================== 4. 初始化嵌入模型（阿里云 text-embedding-v4）====================
embeddings = DashScopeEmbeddings(model="text-embedding-v4")

# ==================== 5. 创建 / 加载 Chroma 向量库 ====================
persist_dir = "./chroma_db"
if not os.path.exists(persist_dir):
    # 首次运行：创建并持久化
    vectordb = Chroma.from_documents(
        documents=chunks,
        embedding=embeddings,
        persist_directory=persist_dir
    )
    print(f"✅ 向量库创建完成，共 {vectordb._collection.count()} 个向量")
else:
    # 后续运行：直接加载
    vectordb = Chroma(
        persist_directory=persist_dir,
        embedding_function=embeddings
    )
    print(f"✅ 加载已有向量库，共 {vectordb._collection.count()} 个向量")

# 创建检索器（k=3 表示每次检索返回最相关的 3 个块）
retriever = vectordb.as_retriever(search_kwargs={"k": 3})

# ==================== 6. 初始化 DeepSeek 模型 ====================
llm = ChatOpenAI(
    model="deepseek-chat",               # DeepSeek 模型名称
    openai_api_key=os.getenv("DEEPSEEK_API_KEY"), # 从环境变量读取密钥
    base_url="https://api.deepseek.com/v1",       # DeepSeek API 地址
    temperature=0.7,                    
)

# ==================== 7. 构建提示模板 ====================
template = """你是一个基于文档内容的问答助手。请根据以下上下文片段回答用户的问题。
如果上下文不足以回答问题，请如实说“根据现有文档无法回答”，不要编造信息。

<上下文>
{context}
</上下文>

用户问题：{question}

请给出准确、简洁的回答："""

prompt = ChatPromptTemplate.from_template(template)

# ==================== 8. 构建 RAG 链（LCEL）====================
def format_docs(question: str) -> str:
    docs = retriever.invoke(question)
    if not docs:
        return "没有找到相关文档片段。"
    return "\n\n".join([doc.page_content for doc in docs])

rag_chain = (
    {
        "context": RunnableLambda(format_docs),
        "question": RunnablePassthrough()
    }
    | prompt
    | llm
)

# ==================== 9. 测试问答 ====================
question = "这份PDF文档核心内容是什么？"
response = rag_chain.invoke(question)
print("\n🤖 回答：", response.content)
```

## 三、调优备忘

### 1. 分块策略（`chunk_size` / `chunk_overlap`）

| 场景             | 推荐参数                                              | 说明                     |
| ---------------- | ----------------------------------------------------- | ------------------------ |
| 短文档、问答型   | `chunk_size=300, overlap=30`                          | 颗粒度细，精准匹配       |
| 长文档、叙述型   | `chunk_size=800, overlap=100`                         | 保留上下文，减少断裂     |
| 技术手册、代码   | `chunk_size=1000, overlap=150`，配合 `LanguageParser` | 保持代码块完整           |
| 检索结果零散     | 增大 `chunk_overlap`                                  | 让关键信息出现在多个块中 |
| 答案包含过多噪音 | 减小 `chunk_size`                                     | 过滤不相关内容           |

**调试验证**：固定几个问题，分别用不同参数重建向量库，观察检索到的 top‑3 片段质量。

### 2. 检索参数（`k` 与检索类型）

- **`k` 值**：通常设为 3～5。  
  - 太小 → 可能遗漏答案  
  - 太大 → 增加 token 消耗，引入噪音  
- **检索类型**：  
  - `similarity`（默认）：基于余弦相似度，简单直接  
  - `mmr`（最大边际相关性）：减少返回结果之间的冗余，适合内容重复度高的文档  
    ```python
    retriever = vectordb.as_retriever(
        search_type="mmr",
        search_kwargs={"k": 4, "fetch_k": 10}
    )
    ```

### 3. 调试技巧：查看源文档

单独调用 `retriever` 查看检索结果，不干扰生成链：

```python
question = "你的问题"
docs = retriever.invoke(question)
for i, doc in enumerate(docs):
    print(f"片段 {i+1}: {doc.page_content[:200]}...")
    print(f"来源: {doc.metadata}\n")
```

若检索结果相关但答案不佳 → 优化 prompt 或降低 `temperature`  
若检索结果不相关 → 调整分割参数、`k` 值或使用 `mmr`

## 四、常见问题与解决方案

| 问题现象                | 可能原因                   | 解决方案                                        |
| ----------------------- | -------------------------- | ----------------------------------------------- |
| 回答明显错误或幻觉      | 检索片段不包含答案         | 增大 `chunk_size` 或 `k`；检查 PDF 解析是否完整 |
| 回答过于啰嗦            | prompt 未约束简洁性        | 修改模板加入“请给出简洁回答”                    |
| 中文支持差              | 分隔符未包含中文标点       | 在 `separators` 中加入 `"。", "，"`             |
| 调用阿里 embedding 报错 | 未设置 `DASHSCOPE_API_KEY` | 检查环境变量或直接在代码中赋值                  |
| DeepSeek 调用超时       | 网络问题或 API 限制        | 增加 `timeout` 参数，或使用代理                 |
