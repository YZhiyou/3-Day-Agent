## 模块三：进阶记忆策略——摘要记忆实战

这一节的目标是：将模块一中你自定义的 `SummaryChatMessageHistory` 与模块二的 `invoke_with_memory` 无缝结合，让 Agent 在长对话中自动压缩历史，有效控制 Token 消耗。

### 3.1 为什么需要摘要记忆？

回顾模块一中全量记忆的问题：每轮对话都会追加新消息，Token 消耗线性增长。一个 50 轮的多工具调用 Agent 对话，历史消息轻松突破上万 Token，不仅成本高昂，还可能超出模型的上下文窗口（如 DeepSeek‑Chat 的 128K，但过多历史会稀释注意力，降低回答质量）。

摘要记忆的核心思想是：**只保留最近的原始消息 + 较早期的摘要**。这样既保留了长篇对话的关键脉络，又大幅减少了 Token 使用。

你在模块一中实现的 `SummaryChatMessageHistory` 已经封装了这套逻辑：
- 内部维护 `_messages` 和 `summary_text`
- `messages` 属性返回 `[SystemMessage(summary_text), ...最近 max_recent 条原始消息]`
- 当 `_messages` 数量超过 `max_recent * 2` 时，自动调用 LLM 生成/更新摘要

现在，我们只需要让 Agent 使用它作为历史存储。

---

### 3.2 集成步骤

由于我们在模块二中设计的手动管理方案完全依赖 `get_session_history` 返回的历史对象，所以**替换记忆策略只需修改这一处**。

#### 步骤①：选择性地为不同会话配置不同策略

我们可以根据 `session_id` 的前缀或自定义规则，返回不同的历史实例：

```python
from langchain_core.chat_history import BaseChatMessageHistory

# 你的 SummaryChatMessageHistory 完整定义（复用模块一代码，此处省略，只要导入即可）
# 假设它已保存在当前文件中，或通过 from my_histories import SummaryChatMessageHistory

store = {}

def get_session_history(session_id: str) -> BaseChatMessageHistory:
    if session_id not in store:
        # 对长会话使用摘要记忆（例如以 "long_" 开头的 session_id）
        if session_id.startswith("long_"):
            store[session_id] = SummaryChatMessageHistory(
                summarization_llm=llm,  # 用于生成摘要的 LLM，可以和 Agent 用同一个模型
                max_recent_messages=4   # 保留最近 4 条原始消息
            )
        else:
            store[session_id] = InMemoryChatMessageHistory()
    return store[session_id]
```

#### 步骤②：`invoke_with_memory` 完全不需要改动

因为它只依赖 `history.messages` 和 `history.add_message`，而 `SummaryChatMessageHistory` 完美实现了这两个接口。

### 3.3 完整可运行代码（摘要记忆版 Agent）

下面是一个完整的例子，展示 Agent 如何在长对话中自动维持摘要记忆。注意这里复用了同一个 `llm` 作为摘要生成器（节省资源），你也可单独使用更便宜的模型。

```python
from langchain.agents import create_agent
from langchain_openai import ChatOpenAI
from langchain_tavily import TavilySearch
from langchain_core.chat_history import InMemoryChatMessageHistory, BaseChatMessageHistory
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage, SystemMessage
from typing import List
from dotenv import load_dotenv
import os

load_dotenv()

# ========== 1. 模型 & 工具 ==========
llm = ChatOpenAI(
    model="deepseek-chat",
    openai_api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url="https://api.deepseek.com/v1",
    temperature=0,
)

search = TavilySearch(
    tavily_api_key=os.getenv("TAVILY_API_KEY"),
    max_results=2,
)
tools = [search]

# ========== 2. 自定义摘要记忆（模块一代码，略有简化但逻辑一致） ==========
class SummaryChatMessageHistory(BaseChatMessageHistory):
    def __init__(self, summarization_llm: ChatOpenAI, max_recent_messages: int = 4):
        self._messages: List[BaseMessage] = []
        self.summary_text: str = ""
        self.summarization_llm = summarization_llm
        self.max_recent = max_recent_messages

    @property
    def messages(self) -> List[BaseMessage]:
        result = []
        if self.summary_text:
            result.append(SystemMessage(content=self.summary_text))
        result.extend(self._messages[-self.max_recent:])
        return result

    def add_messages(self, messages: List[BaseMessage]) -> None:
        self._messages.extend(messages)
        if len(self._messages) > self.max_recent * 2:
            self._generate_summary()

    def add_message(self, message: BaseMessage) -> None:
        self.add_messages([message])

    def _generate_summary(self):
        old_messages = self._messages[:-self.max_recent]
        conv_text = "\n".join(
            [f"{'User' if isinstance(m, HumanMessage) else 'AI'}: {m.content}" for m in old_messages]
        )
        prompt = f"请用一段话总结以下对话内容，保留关键信息：\n{conv_text}\n摘要："
        new_summary = self.summarization_llm.invoke(prompt).content
        if self.summary_text:
            self.summary_text = self.summary_text + "\n" + new_summary
        else:
            self.summary_text = new_summary
        self._messages = self._messages[-self.max_recent:]

    def clear(self) -> None:
        self._messages = []
        self.summary_text = ""


# ========== 3. 历史存储与回调 ==========
store = {}

def get_session_history(session_id: str) -> BaseChatMessageHistory:
    if session_id not in store:
        # 所有会话都使用摘要记忆，方便观察效果
        store[session_id] = SummaryChatMessageHistory(
            summarization_llm=llm,
            max_recent_messages=4   # 设置小一点便于快速触发摘要
        )
    return store[session_id]

def invoke_with_memory(agent, session_id: str, user_content: str):
    history = get_session_history(session_id)
    messages = list(history.messages) + [HumanMessage(content=user_content)]
    response = agent.invoke({"messages": messages})
    # 保存交互
    history.add_message(HumanMessage(content=user_content))
    history.add_message(AIMessage(content=response["messages"][-1].content))
    return response

# ========== 4. 创建 Agent ==========
agent = create_agent(model=llm, tools=tools)

# ========== 5. 模拟长对话，观察摘要生成 ==========
SESSION_ID = "long_demo"

# 前几轮对话（累积到触发摘要）

nvoke_with_memory(agent, SESSION_ID, "我想了解机器学习的基本分类")
invoke_with_memory(agent, SESSION_ID, "主要分为监督、无监督和强化学习，对吗？")
invoke_with_memory(agent, SESSION_ID, "监督学习和无监督学习有什么区别？")
invoke_with_memory(agent, SESSION_ID, "能举个例子吗？用图像识别来讲")    
invoke_with_memory(agent, SESSION_ID, "那半监督学习呢？解释一下")


# 查看历史对象内部状态（调试用）
history_obj = store[SESSION_ID]
print("=== 当前摘要文本 ===")
print(history_obj.summary_text)
print("\n=== 提供的消息（Agent 实际接收的上下文） ===")
for msg in history_obj.messages:
    print(f"{msg.__class__.__name__}: {msg.content[:100]}...")

# 再问一个问题，验证 Agent 能利用摘要
response = invoke_with_memory(agent, SESSION_ID, "我们刚才讨论了哪些学习类型？")
print("\n=== Agent 回答 ===")
print(response["messages"][-1].content)
```

**运行关键现象**：
- 当对话轮数超过 `max_recent * 2 = 8` 条消息时，`_generate_summary` 被触发。
- `summary_text` 中存储了早期对话的压缩信息（如“用户询问了机器学习分类，AI 介绍了监督、无监督、强化学习...”）。
- `messages` 属性返回 `SystemMessage(摘要) + 最近4条原始消息`，Agent 既能从摘要中理解全局背景，又能看到最近的细节。
- Token 数量不再随总轮数线性增长，而是稳定在“摘要长度 + 最近消息长度”的范围内。

### 3.4 窗口记忆同样可以集成

如果你想使用滑动窗口记忆，只需在 `get_session_history` 中换成模块一的 `WindowedChatMessageHistory` 即可：

```python
store[session_id] = WindowedChatMessageHistory(window_size=4)
```

其余代码完全不变，体现了接口兼容的优势。

---

### 3.5 策略选择建议

| 场景               | 记忆策略          | Token 消耗 | 细节保留      |
| ------------------ | ----------------- | ---------- | ------------- |
| 简单问答、调试     | 全量 (`InMemory`) | 线性增长   | 完整          |
| 客服、近期敏感任务 | 窗口 (`Windowed`) | 固定       | 仅近期        |
| 长对话、需全局理解 | 摘要 (`Summary`)  | 缓慢增长   | 压缩+近期细节 |

在真实生产环境中，你还可以组合使用：例如用 **摘要 + 窗口**，即摘要概括早期历史，窗口保留最近几轮原始消息。这正是 `SummaryChatMessageHistory` 设计的思想。
