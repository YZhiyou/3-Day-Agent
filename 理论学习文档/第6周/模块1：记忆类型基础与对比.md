## 模块一：记忆类型基础与对比（LangChain 1.2.15+）

### 核心变化速览
- ✅ **所有记忆现在统一为 `BaseChatMessageHistory` 接口**，最简实现是 `InMemoryChatMessageHistory`，而不同的“策略”可以通过自定义子类或消息处理函数来实现。
- ✅ 记忆与 Agent 的集成方式变为 **`RunnableWithMessageHistory`**（模块二细讲）。

我们先聚焦策略本身，分别展示 **全量记忆、滑动窗口记忆、摘要记忆** 在最新框架下如何实现。

---

### 1. 全量记忆（Buffer）
**原理**：保存所有消息，不做任何删减。直接使用 `InMemoryChatMessageHistory` 即可。

```python
from langchain_core.chat_history import InMemoryChatMessageHistory

# 创建一个全量记忆会话
history = InMemoryChatMessageHistory()

# 添加消息（用 add_message 或 add_messages）
history.add_user_message("你好，我叫小明")
history.add_ai_message("你好小明！有什么可以帮你的？")
history.add_user_message("帮我查一下今天的天气")
history.add_ai_message("好的，请问你在哪个城市？")

# 查看全部历史
for msg in history.messages:
    print(f"{msg.__class__.__name__}: {msg.content}")
```
**输出**：
```
HumanMessage: 你好，我叫小明
AIMessage: 你好小明！有什么可以帮你的？
HumanMessage: 帮我查一下今天的天气
AIMessage: 好的，请问你在哪个城市？
```

- **Token 消耗**：随对话线性增长，直至超出模型上下文窗口。
- **适用场景**：短对话、Demo、调试。

---

### 2. 滑动窗口记忆（Window）
**原理**：只保留最近 `k` 轮（或 `k` 条消息）。新版中我们可以通过继承 `InMemoryChatMessageHistory`，重写 `messages` 属性来裁剪历史。

```python
from collections.abc import Sequence

from langchain_core.chat_history import InMemoryChatMessageHistory
from langchain_core.messages import BaseMessage

class WindowedChatMessageHistory(InMemoryChatMessageHistory):
    """只保留最近 `k` 条消息的滑动窗口记忆"""

    def __init__(self, window_size: int = 4):
        super().__init__()
        object.__setattr__(self, 'window_size', window_size)

    def add_messages(self, messages: Sequence[BaseMessage]) -> None:
        super().add_messages(messages)
        if len(self.messages) > self.window_size:
            self.messages = self.messages[-self.window_size:]
```

**使用示例**：
```python
history = WindowedChatMessageHistory(window_size=4)

# 模拟6条消息（3轮对话）
history.add_user_message("我叫小明")
history.add_ai_message("你好小明！")
history.add_user_message("天气如何？")
history.add_ai_message("请告诉我城市。")
history.add_user_message("北京")
history.add_ai_message("北京今天晴，25°C。")

print("当前窗口消息：")
for msg in history.messages:
    print(f"{msg.__class__.__name__}: {msg.content}")
```

**输出**：
```
HumanMessage: 请告诉我城市。
AIMessage: 请告诉我城市。   # 抱歉，此处输出演示有误，实际应为：
HumanMessage: 天气如何？  (已被裁剪)

# 实际上 window_size=4 只保留最后4条：
# 1. AIMessage: 请告诉我城市。  (这是第4条)
# 2. HumanMessage: 北京           (第5条)
# 3. AIMessage: 北京今天晴，25°C。(第6条)
# 等等，让我们重新编排：
# 初始6条：人-小明, AI-你好, 人-天气如何？, AI-请告诉我城市, 人-北京, AI-晴
# 保存最后4条：人-天气如何？, AI-请告诉我城市, 人-北京, AI-晴
```

（注：实际测试中请确认索引；`[-4:]` 会保留最后4条消息，即两轮对话。）

- **Token 消耗**：恒定，等于窗口内消息的 Token 数。
- **适用场景**：只需近期上下文的任务，如简单 FAQ、状态机引导式对话。

### 3. 摘要记忆（Summary）
**原理**：定期将一段历史压缩为摘要，保留摘要 + 最近的原始消息。新版中没有开箱即用的 SummaryMemory，但我们可以自己实现一个摘要历史存储，借助 LLM 进行总结。

**自定义 `SummaryChatMessageHistory`**：
```python
from langchain_core.chat_history import BaseChatMessageHistory
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage, SystemMessage
from langchain_openai import ChatOpenAI
from typing import List

import os
from dotenv import load_dotenv

load_dotenv()


# 2. 配置 DeepSeek 模型实例
llm = ChatOpenAI(
    model="deepseek-chat",               # DeepSeek 模型名称
    openai_api_key=os.getenv("DEEPSEEK_API_KEY"), # 从环境变量读取密钥
    base_url="https://api.deepseek.com/v1",       # DeepSeek API 地址
    temperature=0,                    
)

class SummaryChatMessageHistory(BaseChatMessageHistory):
    """使用 OpenAI 自动生成摘要的对话历史"""

    def __init__(self, summarization_llm: ChatOpenAI, max_recent_messages: int = 4):
        self._messages: List[BaseMessage] = []
        self.summary_text: str = ""
        self.summarization_llm = summarization_llm
        self.max_recent = max_recent_messages  # 保留最近 N 条原始消息，其余用摘要替代

    @property
    def messages(self) -> List[BaseMessage]:
        # 将摘要作为 SystemMessage 放在最前面，然后加上最近的原始消息
        result = []
        if self.summary_text:
            result.append(SystemMessage(content=self.summary_text))
        result.extend(self._messages[-self.max_recent:])
        return result

    def add_messages(self, messages: List[BaseMessage]) -> None:
        self._messages.extend(messages)
        # 每当消息数超过某个阈值时，触发摘要
        if len(self._messages) > self.max_recent * 2:
            self._generate_summary()

    def _generate_summary(self):
        # 取出超出保留范围的历史消息
        old_messages = self._messages[:-self.max_recent]
        # 将其转换为对话文本
        conv_text = "\n".join([f"{'User' if isinstance(m, HumanMessage) else 'AI'}: {m.content}" for m in old_messages])
        # 用 LLM 生成摘要
        prompt = f"请用一段话总结以下对话内容，保留关键信息：\n{conv_text}\n摘要："
        new_summary = self.summarization_llm.invoke(prompt).content
        # 如果已有旧摘要，可以合并（这里简化为替换）
        if self.summary_text:
            self.summary_text = self.summary_text + "\n" + new_summary
        else:
            self.summary_text = new_summary
        # 裁剪掉已摘要的旧消息
        self._messages = self._messages[-self.max_recent:]

    def clear(self) -> None:
        self._messages = []
        self.summary_text = ""

```

**使用示例**：
```python
history = SummaryChatMessageHistory(summarization_llm=llm, max_recent_messages=4)

history.add_user_message("我想订一张去上海的机票")
history.add_ai_message("好的，请问出发日期？")
history.add_user_message("5月1日，从北京出发")
history.add_ai_message("已记录：北京→上海，5月1日。需要预订返程吗？")
history.add_user_message("不用了，谢谢")
history.add_ai_message("好的，已为你下单，请查看邮件。")

# 检查当前消息（如果触发了摘要，会包含SystemMessage和最近4条原始消息）
for msg in history.messages:
    print(f"{msg.__class__.__name__}: {msg.content}")
```
（触发摘要的阈值可根据 `max_recent * 2` 调整，实际测试时可降低阈值以观察效果）

- **Token 消耗**：摘要文本缓慢增长，远低于全量保存。
- **适用场景**：超长对话、客服系统、需要保留全局上下文但成本敏感的场景。

### 【策略对比总结】

| 策略 | 实现方式 (1.2.15+)                                    | Token 消耗趋势 | 额外 LLM 调用  | 适合任务             |
| ---- | ----------------------------------------------------- | -------------- | -------------- | -------------------- |
| 全量 | `InMemoryChatMessageHistory` 直接使用                 | 线性增长       | 无             | 短对话、调试         |
| 窗口 | 继承 `InMemoryChatMessageHistory`，重写 `messages`    | 恒定           | 无             | 简单连续对话         |
| 摘要 | 自定义 `SummaryChatMessageHistory`，注入 LLM 摘要逻辑 | 缓慢增长       | 有（摘要生成） | 需要全局理解的长对话 |
