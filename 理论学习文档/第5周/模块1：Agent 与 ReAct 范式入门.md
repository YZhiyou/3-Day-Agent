### 模块一：Agent 与 ReAct 范式入门

#### 1. Agent 是什么？它与 Chain 的根本区别

你已经熟练使用 LCEL 构建过复杂 Chain。我们来做一个对比，帮助你建立直觉：

| 特性             | **Chain（链）**                                  | **Agent（智能体）**                                          |
| ---------------- | ------------------------------------------------ | ------------------------------------------------------------ |
| **执行路径**     | 预先定义好的硬编码顺序（如：检索 → 生成 → 输出） | 由模型**自主动态决策**下一步做什么                           |
| **工具调用**     | 如果需要工具，必须由开发者明确写在链的某个位置   | 模型自己判断**是否调用工具、调用哪个工具、按什么顺序调用**   |
| **应对不确定性** | 遇到预料之外的问题容易出错或无法处理             | 可以根据中间结果**调整策略**，例如换一个工具再试             |
| **比喻**         | 工厂流水线：零件按固定工序流转                   | 实习生：给你目标，你决定先查资料还是先计算，查不到再想别的办法 |

> **一句话总结**：Chain 执行的是你预设的“剧本”，Agent 执行的是它自己推理出来的“行动计划”。

#### 2. ReAct 范式：Agent 的思考引擎

LangChain 中最经典的 Agent 架构是 **ReAct**（Reasoning + Acting）。它让模型在**思考**和**行动**之间交替进行，直至得到最终答案。

ReAct 循环包含四个阶段，形成一个闭环：

```
┌─────────────┐
│   Thought   │  ← 模型分析当前情况，决定下一步做什么
└──────┬──────┘
       ▼
┌─────────────┐
│   Action    │  ← 执行具体动作（调用某个工具，传入参数）
└──────┬──────┘
       ▼
┌─────────────┐
│ Observation │  ← 工具返回的结果（例如天气数据、计算结果）
└──────┬──────┘
       │
       ▼
   (如果信息足够，结束循环)
┌─────────────┐
│ Final Answer│  ← 基于所有观察，输出最终回答
└─────────────┘
```

- **Thought**：模型内心独白，例如 *“我需要知道北京的天气，所以应该使用天气查询工具。”*
- **Action**：实际调用工具，例如 `weather_tool("Beijing")`。
- **Observation**：工具返回的原始数据，例如 `"北京当前晴天，气温22℃"`。
- **Final Answer**：循环结束后，模型将 Observation 综合成自然语言回复。

#### 3. 一个极简的 ReAct 运行预览

在深入学习代码之前，我们先从逻辑上看一个例子。假设用户问：

> **“北京今天的温度是多少？如果低于25度，帮我算一下150美元等于多少人民币。”**

一个 ReAct Agent 的内心戏可能是这样的（用 `verbose=True` 时你将在控制台看到类似输出）：

```text
> Entering new AgentExecutor chain...

Thought: 用户想知道北京的温度，我需要先查询天气。
Action: get_weather
Action Input: {"city": "Beijing"}
Observation: 北京当前晴天，气温18℃

Thought: 温度是18℃，低于25度。用户还要求计算150美元等于多少人民币，我需要使用计算器。
Action: calculator
Action Input: {"expression": "150 * 7.2"}  (假设汇率7.2)
Observation: 1080.0

Thought: 我现在有了所有需要的信息：北京温度18℃，150美元约合1080元人民币。
Final Answer: 北京今天18℃，低于25度。150美元按7.2汇率计算，等于1080元人民币。

> Finished chain.
```

请先在这个阶段建立一个心智模型：**Agent 不是一步到位给出答案，而是一步一步推理、调用工具、观察结果、再推理的循环过程。**
