## 第三周学习笔记摘要

### LCEL 核心心智模型

> **LCEL 就像一个“智能工厂的传送带系统”。**
> 数据（原材料）在传送带（管道符 `|`）上流动，每个工位（`Runnable` 组件）只负责一个特定操作。你可以随时在传送带上增加**旁路**（`RunnablePassthrough`）、设置**分拣并行区**（`RunnableParallel`）、插入**自定义加工机器**（`RunnableLambda`）或者安装**自动分流挡板**（`RunnableBranch`）。由于所有工位的“接口规格”完全统一，你可以像拼乐高一样任意重组流水线。

### 关键代码资产

这是一个完整的、具备实用价值的 **“智能内容分类与处理链”** 。它包含了本周学到的 `RunnableLambda`（自定义清洗函数）、`RunnableBranch`（条件路由）和 `.bind()`（参数绑定）。

```python
import os
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnableBranch, RunnableLambda, RunnablePassthrough, RunnableParallel

# 1. 初始化模型 (单一实例)
base_model = ChatOpenAI(model="gpt-3.5-turbo")

# 2. 自定义 Python 函数 (使用 RunnableLambda 包装)
def clean_input_text(input_dict: dict) -> dict:
    """预处理：去除输入文本中的多余空格和换行符"""
    raw_text = input_dict.get("user_query", "")
    cleaned_text = " ".join(raw_text.split())
    return {
        **input_dict,
        "cleaned_query": cleaned_text,
        "word_count": len(cleaned_text.split())
    }

clean_text_runnable = RunnableLambda(clean_input_text)

# 3. 定义不同分支的提示词与链
# 通用解析器
parser = StrOutputParser()

# 分支 A: 简短问候 (高随机性，绑定了 temperature=0.9)
greeting_prompt = ChatPromptTemplate.from_template("用一句热情友好的话回复用户的问候：{cleaned_query}")
greeting_chain = greeting_prompt | base_model.bind(temperature=0.9) | parser

# 分支 B: 技术问题 (严谨精确，绑定了 temperature=0.1)
tech_prompt = ChatPromptTemplate.from_template("作为资深专家，言简意赅且准确地回答技术问题：{cleaned_query}")
tech_chain = tech_prompt | base_model.bind(temperature=0.1) | parser

# 分支 C: 默认通用
general_prompt = ChatPromptTemplate.from_template("回答用户的问题：{cleaned_query}")
general_chain = general_prompt | base_model.bind(temperature=0.5) | parser

# 4. 构建分支条件
def is_greeting(input_dict: dict) -> bool:
    text = input_dict["cleaned_query"].lower()
    return any(w in text for w in ["hello", "hi", "你好", "早上好"])

def is_tech(input_dict: dict) -> bool:
    text = input_dict["cleaned_query"].lower()
    return any(w in text for w in ["python", "代码", "bug", "api", "函数", "类"])

# 5. 最终组装链 (包含并行结果记录)
final_chain = (
    # 第一步：清洗数据
    clean_text_runnable
    # 第二步：并行记录原始信息和执行分支路由
    | RunnableParallel(
        original_input=RunnablePassthrough(),  # 保留原始清洗后的字典
        response=RunnableBranch(
            (is_greeting, greeting_chain),
            (is_tech, tech_chain),
            general_chain
        )
    )
)

# 6. 测试运行
if __name__ == "__main__":
    test_queries = [
        {"user_query": "Hello!  你今天过得怎么样？  "},
        {"user_query": "Python 里的 lambda 函数怎么用？"},
        {"user_query": "今天天气真好。"}
    ]
    
    for query in test_queries:
        print(f"用户输入: {query['user_query']}")
        output = final_chain.invoke(query)
        print(f"处理结果: {output['response']}")
        print(f"[系统日志: 清洗后字数 {output['original_input']['word_count']}]")
        print("-" * 50)
```

### 避坑指南

1.  **`RunnableLambda` 的类型陷阱**：
    - **错误表现**：编写了自定义函数返回一个简单字符串，但下游的 `PromptTemplate` 期望接收一个包含特定键的字典（例如 `{"input": "..."}`）。
    - **避坑方法**：在 `RunnableLambda` 包装的函数内部，务必确保**输出类型与下游组件的输入期望严格一致**。如果不确定，请养成返回完整字典的习惯（如示例中的 `return {**input_dict, "new_key": value}`），这样可以保证数据在链中不丢失。

2.  **在 `RunnableParallel` 中错误地使用普通字典**：
    - **错误表现**：直接在代码里定义 `my_parallel = {"key": some_chain}`，然后试图调用 `my_parallel.invoke()`，导致 `AttributeError: 'dict' object has no attribute 'invoke'`。
    - **避坑方法**：请记住，字典只有在**处于管道符 `|` 的上下文中**（例如 `prompt | {"key": chain} | model`）才会被自动转换。如果你想独立定义一个可调用的并行链对象，必须显式导入并使用 `RunnableParallel(key=some_chain)`。
