### 模块四：综合应用与笔记产出

在本模块中，我们将串联前三个模块的知识，构建一个完整的“任务规划器”示例。然后，我会按照你的要求，生成一份《第二周学习笔记摘要》。

#### 综合示例：完整的“任务规划器”链

这个示例将整合以下内容：
- 使用 `ChatOpenAI` 并设置低 `temperature` 以保证格式确定性。
- 使用 `ChatPromptTemplate.from_messages` 构建包含系统指令和用户输入的结构化提示词。
- 使用 `PydanticOutputParser` 定义任务计划的数据结构，并配合 `OutputFixingParser` 增强容错性。

```python
from pydantic import BaseModel, Field
from typing import List
from langchain_core.output_parsers import PydanticOutputParser
from langchain.output_parsers import OutputFixingParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

# 1. 定义 Pydantic 数据模型
class Task(BaseModel):
    title: str = Field(description="任务的简短标题")
    priority: int = Field(description="任务优先级，1为最高，5为最低", ge=1, le=5)
    estimated_hours: float = Field(description="预计完成所需小时数", ge=0)

class TaskPlan(BaseModel):
    tasks: List[Task] = Field(description="待办任务列表")
    overall_goal: str = Field(description="对用户输入目标的简洁总结")

# 2. 创建基础解析器和修复解析器
base_parser = PydanticOutputParser(pydantic_object=TaskPlan)
fixing_parser = OutputFixingParser.from_llm(
    llm=ChatOpenAI(model="gpt-3.5-turbo", temperature=0.0),
    parser=base_parser
)

# 3. 构建 Prompt 模板
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个专业的任务规划助手。请根据用户的输入，生成一个详细的任务计划。\n{format_instructions}"),
    ("human", "我的目标是：{goal}")
])
prompt = prompt.partial(format_instructions=base_parser.get_format_instructions())

# 4. 创建模型（低温度确保格式遵循）
model = ChatOpenAI(model="gpt-3.5-turbo", temperature=0.0)

# 5. 构建链：prompt -> model -> fixing_parser
chain = prompt | model | fixing_parser

# 6. 测试调用
if __name__ == "__main__":
    user_goal = input("请输入您的目标：")
    try:
        result = chain.invoke({"goal": user_goal})
        print(f"\n总体目标：{result.overall_goal}")
        print("任务清单：")
        for i, task in enumerate(result.tasks, 1):
            print(f"  {i}. {task.title} (优先级 {task.priority}，预计 {task.estimated_hours} 小时)")
    except Exception as e:
        print(f"运行失败：{e}")
```

### 《第二周学习笔记摘要》

#### 核心概念图解

LangChain 中 **Model、Prompt、Parser** 三者构成了经典的数据流闭环：

```
[用户输入] 
    ↓
[Prompt 模板] → 注入系统指令、示例和格式约束
    ↓
[ChatModel] → 根据温度等参数生成文本响应
    ↓
[OutputParser] → 解析文本并校验为结构化数据
    ↓
[Pydantic 对象实例] → 供下游业务逻辑使用
```

- **Model**：负责根据输入生成文本，通过 `temperature` 等参数调节随机性。
- **Prompt**：负责构造完整的上下文，通过 `from_messages`、`MessagesPlaceholder` 等提供丰富的角色与历史信息。
- **Parser**：负责将文本响应转化为强类型数据结构，利用 Pydantic 的 `Field` 描述和约束引导模型输出正确格式。

#### 关键代码资产

**一个完整的、带容错机制的“任务规划器”链**（如上文模块四中的综合示例代码）。该代码展示了如何定义 Pydantic 模型、生成格式指令、构建 Prompt、创建链并使用修复解析器增强健壮性，可直接复制到项目中使用。

#### 避坑指南

1. **`temperature=0.0` 并不能保证 100% 的格式正确性**  
   - **错误认知**：以为只要把温度设为零，模型就一定输出完美 JSON。  
   - **实际情况**：LLM 仍可能偶尔输出额外的解释文字或格式错误。  
   - **解决方法**：在生产代码中务必配合 `OutputFixingParser` 或 `RetryOutputParser` 等容错机制，提高成功率。

2. **忘记在 Prompt 中注入 `format_instructions`**  
   - **错误现象**：链运行成功，但解析器总是抛出 `ValidationError`，因为模型返回的是普通文本而非 JSON。  
   - **根本原因**：没有将 `parser.get_format_instructions()` 放入 Prompt 模板，导致模型根本不知道要输出 JSON。  
   - **解决方法**：使用 `prompt.partial(format_instructions=parser.get_format_instructions())` 将指令固定到模板中，或每次调用时手动传入。
